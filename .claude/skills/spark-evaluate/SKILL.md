# spark-evaluate:评估门

逐条挑战核心假设,出 go/no-go/maybe;no-go 写死因留池,不删除。

---

## 开头必读

激活后先读,再开口:

1. `背景库/_背景索引.md` — 已沉淀的术语与项目背景,避免重复追问已知词。
2. 目标 idea 文件(`想法池/idea-NNNN_*.md`) — 读取 frontmatter + body(重点是 `## 核心假设`)。
3. `规则/死因词表.md` — no-go 时 death_reason 只能取自此表。

读完再开工。

---

## 前置检查

**进入任何 Phase 前,先核查以下条件;不满足立即拦截。**

读目标 idea 的 `research_ref` 字段:

- `research_ref` 为 `null` 或缺失 → **拦截**,回复:
  > 「先 `/调研 idea-NNNN`,没有调研证据没法评估。」

  **不继续执行任何 Phase。**

- `research_ref` 有值(指向调研快照) → 通过,读取 `调研/idea-NNNN_调研.md`,进 Phase 0。

---

## Phase 0–5

### Phase 0 — 读 idea + 调研报告

1. 读 idea 文件全文,提取:
   - `## 核心假设` 中的每一条假设(逐条列出,编号)。
   - frontmatter 里的 `category`、`platform`、`effort`。
2. 读 `调研/idea-NNNN_调研.md` 全文,提取:
   - **只有标 `[已核实]` 的事实**是可采信证据,其余(`[未核实]`/`[AI推测]`/`[存疑]`)仅供参考、不作裁决依据。
   - 竞品清单、价格带、用户痛点、市场空白点、「评估门参考」节。
3. 内部整理「假设 × 证据」对应表,为 Phase 1 做准备。

**此 Phase 只读文件,不向用户输出评估内容。**

---

### Phase 1 — 逐条挑战核心假设

对 `## 核心假设` 里的每一条,给出三值裁决:

| 裁决 | 判定条件 |
|---|---|
| **成立** | 调研报告中有 `[已核实]` 证据直接支持该假设 |
| **存疑** | 无充分 `[已核实]` 证据,既无法支持也无法否定 |
| **证伪** | 调研报告中有 `[已核实]` 证据直接与假设矛盾 |

**硬约束:每条裁决必须引用调研报告里具体的 `[已核实]` 证据(来源 + 事实)。无证据的假设只能判 `存疑`,不得判 `成立` 也不得判 `证伪`。**

输出格式(每条):

```
假设 N:「{假设内容}」
裁决:成立 / 存疑 / 证伪
依据:[已核实] {事实描述} — 来源:{链接/报告章节}
```

---

### Phase 2 — 竞品现实核对(撞车检测)

基于调研报告的竞品清单和市场空白,回答三个问题:

1. **有没有竞品已把这个做得很好**(功能高度重合、用户口碑好、规模大)?
2. **差异化在哪**:这个想法与最强竞品的核心区别是什么?
3. **撞车判定**:若「功能高度重合」且「说不出差异化」→ 判为**撞车**。

撞车判定依据同样只采信 `[已核实]` 数据。差异化的判断若无证据只标 `[AI推测]`。

---

### Phase 3 — 商业快筛

三条快筛,每条给出 通过 / 存疑 / 不通过:

1. **真实付费意愿信号**:调研里有没有用户真实付费的证据(现有竞品定价 + 销量/订阅数)?还是只有"免费+广告"的市场格局?
2. **effort vs 资源**:idea 的 `effort` 估值(S/M/L/XL)与当前可用资源(单人/小团队)是否匹配?effort=XL 而资源 = 单人独立开发者 → 不通过。
3. **平台现实**:平台抽成 / 审核政策 / 算法规则是否会卡死商业路径?(参照 `规则/品类与平台映射.md` 和调研里的平台信息)

---

### Phase 4 — 出 verdict

根据 Phase 1–3 结果,按以下判据出唯一 verdict:

#### `go`

- 核心假设**多数成立**(成立条数 > 存疑+证伪条数)
- **有差异化**(Phase 2 能说清,且非纯 AI 推测)
- effort **可承受**(Phase 3 资源匹配)

→ `verdict: go`、`status: candidate`

#### `maybe`

- 关键假设有**存疑**但未被证伪(缺数据,非方向错)
- 差异化说得出但证据不足
- 快筛无明显致命项

→ `verdict: maybe`、`status: evaluated`
→ 提示:「关键假设缺证据,建议回 `/调研 idea-NNNN` 补数据,或回 `/完善 idea-NNNN` 调整方向。」

#### `no-go`

以下**任一**条件满足即为 no-go:
- 核心假设被**证伪**(有 `[已核实]` 证据直接否定)
- Phase 2 判为**撞车无差异**
- effort **远超资源**且无法拆解

→ `verdict: no-go`、`status: evaluated`、**必填 `death_reason`**

---

### Phase 5 — 写评估快照 + 回填 + 更新总览

#### 5a. 写评估快照

写入 `决策档案/idea-NNNN_评估_YYYYMMDD.md`(结构见下方「评估快照结构」)。

**写完不改**——若日后重新评估,另起新文件带新日期。

#### 5b. 回填 idea frontmatter

严格按 `规则/schema与阶段定义.md` 字段名:

```yaml
verdict: go / no-go / maybe
eval_ref: 决策档案/idea-NNNN_评估_YYYYMMDD.md
status: candidate   # go 时;otherwise: evaluated
death_reason: {死因词}   # no-go 时必填,取自规则/死因词表.md;其余为 null
updated: {今天日期}
```

#### 5c. 更新状态总览

在 `想法池/_状态总览.md` 对应行更新:`status`、`verdict`、`updated`。

---

## verdict 判据(速查)

| verdict | 核心条件 | status 流转 |
|---|---|---|
| `go` | 假设多数成立 + 有差异化 + effort 可承受 | `evaluated → candidate` |
| `maybe` | 关键假设缺证据(非证伪) | 保持 `evaluated` |
| `no-go` | 假设证伪 / 撞车无差异 / effort 远超资源 | 保持 `evaluated`,**必写 death_reason** |

---

## 死因机制(核心)

no-go 时,`death_reason` **必须**取自 `规则/死因词表.md` 受控词表:

| 死因词 | 适用场景 |
|---|---|
| `缺流量入口` | 产品做得出但没人知道,无获客路径 |
| `缺变现路径` | 有用户但无付费场景或付费意愿极低 |
| `缺技术能力` | 实现门槛超出当前可用技术资产 |
| `撞车无差异` | 竞品已做得很好且想法无差异化 |
| `需求太弱` | 痛点不够痛,调研显示用户不愿买单 |
| `effort过大` | 投入远超单人/小团队可承受范围 |
| `平台限制` | 平台政策/抽成/审核卡死商业路径 |

**为什么必须结构化:** `death_reason` 是 `/找火花` 做互补匹配的燃料。「缺流量入口」死的想法,可与「自带流量渠道」的想法配成 combo;「缺变现路径」死的,与「有付费场景」的配。结构化死因 = 精确的匹配键。

在评估快照里用一句话展开死因细节(不只是词,而是「为什么这条死因落在这个想法上」)。

**绝不删除 no-go 原子**——它留在想法池里,等 `/找火花` 激活。

```markdown
| # | 假设 | 裁决 | 引用的已核实证据 |
|---|---|---|---|

- 最强竞品:{名称}({一句话描述})

- 付费意愿信号:{通过/存疑/不通过} — {一句话依据}

**verdict: {go/no-go/maybe}**

{一句话说明为什么是这个 verdict}

```

## 收尾

Phase 5 完成后,向用户报告:

**go:**
```
评估完成:idea-NNNN「{标题}」→ verdict: go → candidate。
核心假设 {N} 条全部/多数成立,差异化:{差异点}。
下一步:/商业 idea-NNNN 出商业设计(定位/模式/定价/GTM),再 /prd 出设计文档。
```

**no-go:**
```
评估完成:idea-NNNN「{标题}」→ verdict: no-go。
已降级留池,death_reason: {死因词}。
以后 /找火花 时它是燃料,会被用于互补匹配。
```

**maybe:**
```
评估完成:idea-NNNN「{标题}」→ verdict: maybe。
关键假设「{哪条}」缺充分证据,建议:
- 补证据 → /调研 idea-NNNN(指定补查方向)
- 改方向 → /完善 idea-NNNN
```

---

## 防幻觉约束

- **只采信 `[已核实]` 证据**做裁决,`[未核实]`/`[AI推测]` 不作为裁决依据。
- 假设无对应已核实证据时,判 `存疑`,不判 `成立` 也不判 `证伪`。
- 不脑补竞品数据——若调研报告没有,就说「调研报告无对应已核实数据」。

中文进中文出。用户用英文输入 → 对话用英文,但评估快照各节标题和 body 保持中文。
