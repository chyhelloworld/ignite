# spark-background:背景调查

标记不懂的术语/项目 → 征得同意 → 公开联网查 / 私有问你 → 沉淀进 `背景库/`。

---

## 开头必读

激活后**先读 `背景库/_背景索引.md`**,记录哪些术语和项目已沉淀。已在索引里的不重复查、不追问——直接跳过。读完再开口。

---

## Phase 0–4(核心流程)

### Phase 0 — 读背景索引

- 读 `背景库/_背景索引.md` 全文。
- 记下 `## 术语` 表中已有的术语,以及 `## 项目` 表中已有的项目名。
- **不开口**,内部备好清单后进 Phase 1。

---

### Phase 1 — 识别不确定项

扫描当前对话(或指定的 idea 文件)里所有**不确定/未知的名词与项目背景引用**,分为两类:

| 类型 | 说明 | 示例 |
|---|---|---|
| **公开术语** | 行业词汇、产品名、技术名词,可联网查到权威定义 | SteamDB、roguelike、ARR |
| **私有背景** | 疑似用户自己的项目、代号、缩写,只有用户知道 | "我那个副业项目"、"XX 平台"、"那个 App" |

**规则:**
- 已在 Phase 0 索引里的 → 跳过,不列入。
- 拿不准是公开还是私有 → 当私有处理(问用户,不猜)。
- 纯通用词(如"APP""用户""产品")→ 不标记。

---

### Phase 2 — 标记 + 征求同意(D10 硬约束)

**未经用户确认,绝不联网搜索,绝不展开追问。**

把 Phase 1 识别出的项统一列给用户,示例格式:

> 「我有几个词不太确定,动手前先问你:
>
> - **SteamDB**(公开术语,要我联网查个简短定义吗?)
> - **roguelike**(公开术语,要我联网查吗?)
> - **那个副业项目**(像是你的私有项目,能简单说说吗?)
>
> 哪几个需要查/补充?」

**等用户明确点头(文字同意)后,才进入 Phase 3。** 用户说「跳过」或「不用查」的项,直接放行、不写入。

---

### Phase 3a — 公开术语(同意后执行)

对用户同意查的每个公开术语:

1. 用联网搜索(web_search + web_reader)获取定义 + 关键事实。
2. **每条结果必须带来源 URL**;搜不到 URL 的标 `[未核实]`,不用模型记忆冒充。
3. 往 `背景库/概念词典.md` 追加一行,格式与现有表头**完全一致**:

   ```
   | 术语 | 一句话定义 | 关键事实 | 来源 | 日期 | 标签 |
   ```

   - **来源**:填原始 URL;搜不到则写 `无` 并在事实前加 `[未核实]`。
   - **日期**:今天 `YYYY-MM-DD`。
   - **标签**:2–4 个关键词(如 `游戏平台`、`数据库`、`SaaS`)。
   - 事实有来源链接 → 标 `[已核实]`。

4. **一术语一行**,不合并、不拆分。

---

### Phase 3b — 私有背景(同意后执行)

对用户同意补充的每个私有项目,依次问以下 3 点(一次一个,等回应再问下一个):

1. 「这是个什么项目(一句话)?」
2. 「现在到哪一步了?」
3. 「它和当前这个想法的关联是什么?」

收集完后:

- 扫 `背景库/项目档案/` 目录,找已有文件的最大序号 `NNN` → 新序号 = 最大+1(首个用 `001`)。
- 创建 `背景库/项目档案/proj-NNN_名字.md`,结构如下:

  ```markdown
  # {项目名}

  > 来源标签:[本人] · 日期:{YYYY-MM-DD}

  ## 是什么
  {用户原话或整理后描述} [本人]

  ## 现状
  {进展} [本人]

  ## 约束与相关人
  {已知限制、关键人(可用代号)} [本人]

  ## 与当前想法的关联
  {用户说明的关联} [本人]
  ```

- 所有内容标 `[本人]`,**不替用户编造任何内容**。
- 敏感名称可由用户用代号代替,档案里用代号记录。

---

### Phase 4 — 更新索引 + 回归对话

1. **更新 `背景库/_背景索引.md`**:
   - 新公开术语 → 追加到 `## 术语` 表(`| 术语 | 一句话 |`),一句话取概念词典同条。
   - 新私有项目 → 追加到 `## 项目` 表(`| 项目 | 一句话 | 档案 |`),档案填相对路径 `项目档案/proj-NNN_名字.md`。

2. **回到原对话**,带着已补全的背景继续:
   - 若是被 `/完善` 调用过来的 → 提示:「背景已更新,回 `/完善` 继续。」
   - 若是被 `/捕捉` 调用过来的 → 提示:「背景已记录,可继续 `/捕捉` 或 `/完善`。」
   - 若是用户直接 `/背景` 触发的 → 简报本次写入了什么,提示下一步。

---

## 触发约定

| 触发方式 | 说明 |
|---|---|
| 用户手动 `/背景` 或 `/background` | 直接对当前对话进行识别 + 标记流程 |
| `/完善` 内置背景钩子触发 | 完善中遇不确定词,暂停完善,跳入本 skill |
| `/捕捉` 留下 `待补背景:[词]` 标记 | 捕捉阶段不打断,事后由用户手动 `/背景` 或 `/完善` 时触发 |
| 用户说「这几个词你懂吗」「先查清楚这几个词」 | 同手动 `/背景` |

**核心约束(D10):未经用户同意,不联网、不长篇追问。** Phase 2 的确认不可跳过。

---

## 写入规范

### 概念词典(`背景库/概念词典.md`)

- 追加到表尾,**不修改已有行**。
- 列顺序与表头完全一致:`| 术语 | 一句话定义 | 关键事实 | 来源 | 日期 | 标签 |`。
- 一术语一行;「一句话定义」≤ 30 字;「关键事实」1–3 条用分号分隔。
- 无来源 URL → 事实开头加 `[未核实]`,来源列填 `无`。

### 项目档案(`背景库/项目档案/proj-NNN_名字.md`)

- 包含固定节:「是什么 / 现状 / 约束与相关人 / 与当前想法的关联」。
- 全部内容标 `[本人]`,不推断、不扩写。
- 文件名序号三位补零(`proj-001_...`)。

## 语言

中文进中文出。用户用英文输入 → 英文对话,但写入背景库的文件保持中文节标题。
