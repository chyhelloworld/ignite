## Task 1: spark-business(商业设计)

**Files:** Create `.claude/skills/spark-business/SKILL.md`

- [ ] **Step 1: 写 SKILL.md**

Frontmatter(EXACT):
```yaml
---
name: spark-business
description: |
  想法引擎·商业设计。给通过评估门的候选出定位/客群/商业模式/定价/GTM/风险。
  触发:/商业、/business、「出个商业方案」
  Spark business: turn a go-verdict candidate into positioning, model, pricing, and GTM.
---
```

正文必含章节(清晰中文,dbskill 式直接):
1. `# spark-business:商业设计` + 一行职责:只给 `verdict: go` 的候选做商业方案;不写 PRD、不做技术。
2. `## 前置检查` — 读 idea frontmatter:`verdict` 须为 `go`(status=candidate)。否则拦:「这个想法还没过评估门(verdict≠go)。先 `/评估 idea-NNNN`。」
3. `## 开头必读` — 读 `背景库/_背景索引.md`、idea 文件、`调研/idea-NNNN_调研.md`、`决策档案/idea-NNNN_评估_*.md`(取最新)。
4. `## Phase 0–7`:
   - Phase 0:读 idea + 调研(只采信 `[已核实]`)+ 评估结论。
   - Phase 1:**定位**——一句话价值主张(为谁 / 解决什么痛 / 凭什么是你)。
   - Phase 2:**目标客群**——用户画像 + 触达路径(参照 platform)。
   - Phase 3:**商业模式**——用印钞机视角:input(要投入什么)/ output(稳定产出什么)/ 可替代性;怎么赚钱。
   - Phase 4:**定价**——借「定价即产品」:引流款/利润款分层 + 价差检验(理想 10 倍、5–15 倍区间);用调研价格带([已核实])做锚,说明定价依据。
   - Phase 5:**GTM(冷启动 + 获客)**——按 platform 给冷启动路径与获客渠道;借「流量≠收入」提醒别只追量。
   - Phase 6:**风险复核**——列关键商业风险,并回扣评估快照里的核心假设(哪些仍存疑)。
   - Phase 7:**写盘**——写 `产出/idea-NNNN_商业设计.md`(结构见下);idea frontmatter `artifacts` 追加该路径、`updated` 更新、`status` 保持 `candidate`;更新 `想法池/_状态总览.md` 的 updated;推荐 `/prd idea-NNNN`。
5. `## 商业设计文档结构`(写进 `产出/idea-NNNN_商业设计.md`):
   ```markdown
   ---
   idea: idea-NNNN
   date: YYYY-MM-DD
   ---
   # idea-NNNN 商业设计
   ## 定位(一句话价值主张)
   ## 目标客群
   ## 商业模式(input / output / 怎么赚钱)
   ## 定价(引流款 / 利润款 / 价差检验)
   ## GTM(冷启动 + 获客渠道)
   ## 关键风险与待验证假设
   ```
6. `## 公理与证据纪律` — 借商业公理(定价即产品 / 印钞机 / 流量≠收入)作分析框架,公理推出的结论标 `[AI结论]`;定价/体量等事实只引调研 `[已核实]`,无依据标 `[AI推测]`,不脑补。
7. `## 语言` — 中文进中文出。

Non-negotiable:
- 只对 `verdict: go` 候选执行;否则拦截。
- 定价必须引用调研价格带([已核实])作锚,不凭空定价。
- 收尾把产出路径写进 `artifacts`,更新状态总览;`status` 保持 `candidate`(不提前置 documented)。
- 字段名严格匹配 `规则/schema与阶段定义.md`。

- [ ] **Step 2: 校验** — `node tools/validate.mjs` → OK exit 0。
- [ ] **Step 3: dry-run(纸面)** — 对一个 verdict=go 的候选追踪:前置通过 → 6 段产出 → 写 `产出/...商业设计.md` → artifacts 追加 → 推荐 /prd。确认非 go 候选被拦。
- [ ] **Step 4: 提交**
```bash
git add .claude/skills/spark-business/SKILL.md
git commit -m "feat: spark-business 商业设计 skill"
```

## Task 2: spark-prd(PRD/设计文档)

**Files:** Create `.claude/skills/spark-prd/SKILL.md`

- [ ] **Step 1: 写 SKILL.md**

Frontmatter(EXACT):
```yaml
---
name: spark-prd
description: |
  想法引擎·PRD。把候选 + 商业设计转成 PRD（含技术方案概要与可执行任务清单）。
  触发:/prd、「出个 PRD」「写设计文档」
  Spark PRD: turn a candidate + business design into a verifiable PRD with tech outline and task list.
---
```

正文必含章节:
1. `# spark-prd:PRD` + 一行职责:把候选 + 商业设计转成可验证的 PRD;MVP 导向,范围克制。
2. `## 前置检查` — idea 的 `artifacts` 须含商业设计文件(`产出/idea-NNNN_商业设计.md`)。没有 → 拦:「先 `/商业 idea-NNNN` 出商业设计,再写 PRD。」
3. `## 开头必读` — 读 idea、`产出/idea-NNNN_商业设计.md`、`调研/...`、`决策档案/...评估...`,保持与商业设计的定位/定价一致。
4. `## Phase 0–4`:
   - Phase 0:读全部上游(商业设计 / 调研 / 评估)。
   - Phase 1:**PRD 主体**——背景与目标 / 用户与场景(用户故事) / 功能需求(按 MoSCoW 或 P0·P1·P2,**每条带验收标准**) / 非功能需求 / **明确不做的范围**。
   - Phase 2:**技术方案概要**——技术选型 + 系统结构(组件/数据流) + 关键数据模型 + 主要技术风险(轻量、MVP 导向,不过度设计)。
   - Phase 3:**里程碑与任务清单**——里程碑切分 + 可执行 task list(承接用户 development-workflow 的 PRD/架构/任务拆分习惯)。
   - Phase 4:**写盘**——写 `产出/idea-NNNN_PRD.md`(结构见下);idea frontmatter `artifacts` 追加、`status` 置 `documented`、`updated` 更新;更新 `想法池/_状态总览.md`(status=documented)。
5. `## PRD 文档结构`(写进 `产出/idea-NNNN_PRD.md`):
   ```markdown
   ---
   idea: idea-NNNN
   date: YYYY-MM-DD
   based_on: 产出/idea-NNNN_商业设计.md
   ---
   # idea-NNNN PRD
   ## 背景与目标
   ## 用户与场景(用户故事)
   ## 功能需求(P0 / P1 / P2,每条含验收标准)
   ## 非功能需求
   ## 不做的范围(YAGNI)
   ## 技术方案概要(选型 / 结构 / 数据模型 / 风险)
   ## 里程碑与任务清单
   ```
6. `## 一致性与克制` — 定位/定价/客群必须与商业设计一致(冲突则以商业设计为准并标注);需求**可验证**(每条有验收标准);范围克制,先 MVP。
7. `## 语言` — 中文进中文出。

Non-negotiable:
- 无商业设计 artifact 不写 PRD(前置拦截)。
- 每条功能需求**必须有验收标准**;明确列"不做的范围"。
- 收尾把 PRD 路径写进 `artifacts`、`status` 置 `documented`、更新状态总览。
- 与商业设计保持一致,冲突以商业设计为准。

- [ ] **Step 2: 校验** — `node tools/validate.mjs` → OK exit 0。
- [ ] **Step 3: dry-run(纸面)** — 对已有商业设计的候选追踪:前置通过 → PRD 7 节 → status=documented → artifacts 追加。确认无商业设计的候选被拦。
- [ ] **Step 4: 提交**
```bash
git add .claude/skills/spark-prd/SKILL.md
git commit -m "feat: spark-prd PRD skill"
```
